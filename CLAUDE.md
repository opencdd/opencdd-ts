# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working
with code in this repository.

## What this repository is

`@opencdd/opencdd` — the TypeScript port of the Ruby
[`opencdd`](https://github.com/opencdd/opencdd-ruby) gem. Reads CDD
data (CDDAL + YAML) and outputs CDDAL. Shared between the
[OpenCDD Browser](https://github.com/opencdd/opencdd.github.io) and the
OpenCDD Editor.

See
[`opencdd/opencdd.github.io` TODO.astro/00](https://github.com/opencdd/opencdd.github.io/blob/main/TODO.astro/00-architecture-and-decisions.md)
for the multi-repo architecture this package fits into.

## What's in here

- `src/models/` — value objects + Database + tree walkers + utilities.
- `src/validators/` — Runner + R01–R14 rules. Re-exports a few model
  symbols for convenience (Condition), which is why the package's
  root index.ts exports validators as a namespace.
- `src/cddal/` — CDDAL format parser/serializer.
- `src/parcel/` — Parcel sheet schema.
- `src/exporters/` — Json/Yaml/Mermaid/Csv exporters.

`src/models/PropertyIds.generated.ts` and
`src/models/MetaClasses.generated.ts` are GENERATED by the Ruby gem
via the cdd-data submodule (see TODO.astro/02). Never hand-edit.

## Conventions

Inherited from `~/.claude/CLAUDE.md`:

- **NEVER hand-roll serialization** — use the model layer's factories
  and let the framework handle (de)serialization.
- **No `double()` in tests** — use real model instances.
- **Strict TypeScript** with `noUncheckedIndexedAccess`. Regex match
  groups and array access need `?? fallback` or `!`.

Package-specific:

- **Extensionless imports** (e.g. `import { Foo } from "./Foo"`) —
  ESM with `moduleResolution: Bundler`. Consumers must use a bundler
  (Vite, Webpack, esbuild, Rollup). Pure-Node consumers without a
  bundler are not currently supported; revisit when needed.
- **Generated files are read-only at this layer** — they come from
  cdd-data via submodule writes. See TODO.astro/02.

## Publish

Trusted publishing via GitHub OIDC:

1. **npm side (one-time)**: configure
   `@opencdd/opencdd` to accept GitHub Actions OIDC tokens from
   `opencdd/opencdd-ts` `.github/workflows/release.yml`. Done in
   the npm web UI by the package owner.
2. **GitHub side**: the publish workflow (`.github/workflows/release.yml`)
   has `id-token: write` permission and runs
   `npm publish --provenance --access public` on tag push.
3. **To release**: push a tag (`v0.1.0`, `v0.2.0`, ...) — the
   workflow handles the rest.

No `NODE_AUTH_TOKEN` secret is needed (or wanted) — trusted publishing
uses OIDC, not a long-lived token.

## Editing the model layer

This package is a port of the Ruby gem. When the gem's semantics
change, this package must follow. The reverse is not true — this
package can add TS-only conveniences that don't affect the wire shape.

When adding a new value object or method:

1. Add the implementation in `src/models/` (or `validators/`).
2. Export it from the relevant `index.ts`.
3. Add a spec under `tests/`.
4. Bump version per semver; tag for release.
